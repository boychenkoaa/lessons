## Задача 11. Архитектура на монадах состояний

- Слои: функции из pure_robot (`move`, `turn`, ...)=> монадные обертки (`move_cmd`, `turn_cmd`, ...,) => `bind / init / pipe`
- До `bind()` и `init()` додумался сам, про `pipe()` прочитал на хабре
- А вот как обернуть обычные функции в "монадные" обертки -- подсказал ИИ. Сейчас получается, что значение всегда None, хотя задел на "честное" протаскивание оставлен (тип A -- для значения).
- Понял, зачем таки нужны монады. Они снимают необходимость в явном протаскивании состояния через цепочку вычислений. И через это явно разделяют декларируемые вычисления от цепочки присваиваний.
