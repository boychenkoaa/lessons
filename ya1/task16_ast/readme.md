# AST

- "Обычное" AST я как-то делал, а с таким еще не сталкивался.
- Сложное, но интересное задание
- Связи в AST делал через next, сама структура дерева явно не хранится
- Ветвление в дереве (это же все таки дерево!) реализуется в моем случае через функции-обертки команд для робота. Одну функцию сделал явной -- `switch2water_and_move` -- и одну лямбду. 
- `switch2water_and_move` переключается на воду, если нет пены, и пытается идти. По сути, вставляет еще 1 неявный вызов `Мove` в программу. Я не уверен, что это хорошо, но вполне возможно в данной архитектуре...
- Класс команды содержит метод `do()` для внутренней логики и `interpret()` для уже полной обработки контекста. Внутри `do` доступа к логу нет.

- [main.py](main.py) - smoke-тест на 2 случая ветвления
