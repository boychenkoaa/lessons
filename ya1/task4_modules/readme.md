# Задание 4: разбиение на модули
- Я не придерживался строго правила "1 класс = 1 модуль", с моими классами команд это избыточно.
- Но в целом, конечно, удобно, когда модуль представляет собой логически замкнутую систему с понятным API. Получаются уже слои - маленькие наборы классов.

## Модули - слои

### Робот
[robot.py](robot.py)
- Внутренний уровень -- сам робот.
- Содержит классы `Robot` и перечисление `RobotTool`.

### Команды
[cmd.py](cmd.py)
- Содержит общего предка команд -- `RobotCmd` и конкретные команды-наследники
  `MoveRobotCmd`, `TurnRobotCmd`, `SetToolRobotCmd`, `StartRobotCmd`, `StopRobotCmd`.

- Каждая команда может воссоздать себя из списка строк методом `from_args` — разбор и валидацию аргументов, и внешним образом `do(robot)` произвести действие над роботом.

- Переходный слой между интерпретатором, который работает исключительно со строками, и самим роботом, реализующим "низкоуровневые" операции.

### Интерпретатор
[interpret.py](interpret.py)

- Тут один класс -- `Interpreter`.
- Парсит строку, создает команду по их названию и запускает исполнение.
- Не знает деталей реализации конкретных команд.
